Ch03 테스트 코드 작성 순서

테스트 코드 작성 순서
1. 진행해야할 테스트 목록을 작성한다.
2. 쉬운 것부터 진행한다
-> 모든 조건이 통과하는 경우가 쉬운 경우가 된다.
-> 왜냐하면 일치하지 않는 조건들을 고려하지 않아도 되기 때문
-> 쉬운것부터 진행한 후 어려운 것을 진행한다.
3. 예외상황(nullPointException etc)을 먼저 처리한다.
4. 코드로 작성이 어려운 경우 상수로 먼저 처리한 뒤 나중에 코드로 변경한다.
5. 테스트 과정에서 새로운 테스트 사례를 발견하면 목록에 추가하여 진행한다.
6. 리팩토링은 짧은 흐름으로 진행하며, 크기가 클 경우 다음에 진행한다.
